Write clear technical docs in a relaxed, friendly voice.

Keep it short:
- Prefer a 1-3 word title
- Add a 5-10 word description that does not start with "The"
- Keep paragraphs to 1-2 sentences

Structure well:
- Use short imperative section titles
- Avoid repeating the page title in section headings when possible

Be practical:
- Focus on why, when, and how to use something
- Use concise examples over long explanation
- In JS or TS snippets, omit unnecessary trailing semicolons and commas

If writing a commit message for docs, prefix it with `docs:`
